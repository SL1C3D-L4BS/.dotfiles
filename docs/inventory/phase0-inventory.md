### 2. Daemon Inventory (Session-Scope)

#### 2.1 Hyprland `exec-once` daemons (`~/.config/hypr/autostart.conf`)

Current autostart entries (order preserved):

1. `chezmoi apply` — re-applies dotfiles on every session start.
2. `hyprpolkitagent` — Polkit agent as compositor child.
3. `/usr/lib/gnome-keyring-daemon --start --components=secrets,pkcs11,ssh` — secrets + PKCS#11 + SSH agent.
4. `dbus-update-activation-environment --systemd WAYLAND_DISPLAY XDG_CURRENT_DESKTOP=Hyprland`
5. `sleep 1 && systemctl --user start xdg-desktop-portal-hyprland xdg-desktop-portal`
6. `swww-daemon --format xrgb`
7. `sleep 0.5 && waypaper --restore`
8. `sleep 2 && quickshell`
9. `sleep 3 && ags run -d ~/.config/ags`
10. `swaync`
11. `wl-paste --watch cliphist store`
12. `hypridle`
13. `hyprsunset -t 5000`

Notes:

- **Sleep-based ordering** is currently relied on for portals, wallpaper, quickshell, and AGS startup.
- All durable session daemons (bar, notifications, clipboard history, idle, portals starter, wallpaper) are compositor children rather than systemd user services.

### 4. Ownership Inventory (By Artifact Class)

High-level ownership for key artifacts:

- **chezmoi-owned (source of truth in repo):**
  - All `dot_config/hypr/*.conf` (except certain runtime-generated overlays, see below).
  - `dot_config/systemd/user/{atuin,kanshi,restic-backup,openclaw-gateway}.service`.
  - `dot_config/quickshell/**` (bar QML and theme).
  - `dot_config/swaync/**`.
  - `dot_config/ghostty/**`.
  - `dot_config/ags/**`.
  - `dot_config/SL1C3D-L4BS/**` (editions, system-config installers, scaffolds, docs, security notes, OpenClaw templates).

- **Generated / runtime-owned (not present as chezmoi-managed source):**
  - `~/.config/hypr/edition.conf` — edition selection generated by `sl1c3d-edition` tooling.
  - `~/.config/hypr/host.conf` — host/arch hints and overrides (not backed by a static chezmoi file).
  - `~/.openclaw/openclaw.json` and `~/.openclaw/workspace/*.md` — generated/managed by OpenClaw installer and user edits.
  - `~/.local/share/atuin/**`, `~/.local/share/zellij/**`, `~/.local/state/nvim/**` — tool-local state.

- **System-owned:**
  - `/usr/lib/gnome-keyring-daemon`, `dbus-broker`, `xdg-desktop-portal*` units, `pipewire*` units, etc. — managed by system packages, not by chezmoi.

This yields a simple rule for Phase 0: **Hyprland core configs, Quickshell, swaync, Ghostty, and AGS are chezmoi-owned; edition and host overlays plus tool state are generated.**

### 7. Startup Sequence (Current)

Approximate logical order on a successful graphical login into Hyprland:

1. Display manager / TTY launches Hyprland session.
2. Hyprland loads core configs and `autostart.conf`.
3. `exec-once = chezmoi apply` runs, re-applying dotfiles.
4. `hyprpolkitagent` starts as compositor child.
5. `gnome-keyring-daemon` (secrets, PKCS#11, SSH) starts.
6. `dbus-update-activation-environment` pushes `WAYLAND_DISPLAY` and `XDG_CURRENT_DESKTOP=Hyprland` into the user systemd environment.
7. After 1s sleep, `systemctl --user start xdg-desktop-portal-hyprland xdg-desktop-portal`.
8. `swww-daemon` starts, then `waypaper --restore` restores wallpaper after 0.5s.
9. `quickshell` starts after 2s.
10. `ags` starts after 3s.
11. `swaync` starts.
12. `wl-paste --watch cliphist store` starts clipboard history daemon.
13. `hypridle` starts (idle + lock orchestration).
14. `hyprsunset -t 5000` applies daytime color temperature.

This sequence is **heavily dependent on `exec-once` and sleeps** rather than systemd user units and explicit target ordering.

### 8. Benchmark Surfaces (For Baselines)

Benchmarks captured in Phase 0:

- **Shell startup (interactive zsh):**
  - Command: `time zsh -i -c 'exit'`
  - Result (single run): **0.079s total** (0.05s user, 0.03s system).

- **Neovim startup:**
  - Command: `nvim --headless --startuptime /tmp/nvim-startup.log +qall`
  - Result (from log): **~45.658 ms** to `--- NVIM STARTED ---`.

- **Config validation (`validate-configs.sh`):**
  - Command: `time ~/scripts/validate-configs.sh >/tmp/validate-configs.log 2>&1`
  - Result (single run): **0.685s total** (0.28s user, 0.16s system).

These values serve as **Phase 0 baselines** and will be formalized with reproducibility rules in Phase 5.
